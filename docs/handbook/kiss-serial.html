Serial KISS TNC

Configure graywolf to use hardware TNC connected as a serial port

There are several types of devices which provide an external KISS TNC:

  • Hardware TNC devices like the Mobilinkd
  • Handheld radios like the Kenwood TH-D75 or VGC VR-N76 which expose a KISS TNC via a cable or bluetooth

The software TNC in graywolf probably outperforms these external TNCs, so you might consider revising your configuration to connect your radio to graywolf using an audio interface. But you can still use these external devices with graywolf.

Graywolf does not yet have support for serial KISS TNCs, so to make it work we'll need a small helper program. Download the free tnc-server program. This simple software will talk to your serial KISS TNC, and expose it as a network server over TCP. You can then configure a KISS interface of type TCP Client in graywolf which talks to the `tnc-server` program.

You don't need any additional hardware for this to work, the `tnc-server` program will comfortably run on the same computer that graywolf is running on. Download the latest release of tnc-server for your operating system and architecture, it supports linux, macOS, and Windows. The README has all the details of how it works.

Here's an example of how to use `tnc-server` to connect graywolf to the KISS TNC in a Kenwood TH-D75. On Linux, a Kenwood TH-D75 often shows up as `/dev/ttyACM0`. For most hardware TNCs the baud rate doesn't matter, I use 9600 baud. Start `tnc-server` as: 

tnc-server -port /dev/ttyACM0 -baud=1200 -listen=localhost:6700

Now create a Channel called "TH-D75" that is a KISS TNC only channel. After that, create a KISS TNC Interface of type `TCP Client`, and tell it to connect to `localhost` port `6700`.
